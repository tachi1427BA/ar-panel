# AR Character Viewer

AR でキャラクター立ち絵を表示するシンプルな Web アプリです。  
A-Frame と 8th Wall Engine (XR8) を使い、**iOS Safari・Android Chrome** の両方でキャラクター画像を実空間の平面に配置できます。

## 概要

- プリセット画像からキャラクターを選択できる
- ローカル画像をアップロードして差し替えできる
- **iOS Safari および Android Chrome** で実空間の平面に複数キャラクターを配置できる
- 非対応環境では 3D ビューにフォールバックして表示できる
- 配置後はタップで編集対象を選び、拡大・縮小できる
- UI を完全に隠す撮影モードを利用できる
- 同じキャラクターは重複配置されず、再配置しようとすると既存の同キャラクターが選択される

## 現在の構成

ES Modules を使った複数ファイル構成です。**ES Modules の仕様上、`file://` で直接開くと動作しません。必ずローカル HTTP サーバー経由でアクセスしてください。**

### ローカルサーバーの起動方法

```bash
# Node.js がある場合
npm start
# または
npx serve .

# Python がある場合
python3 -m http.server 8080
```

起動後 http://localhost:8080 をブラウザで開いてください。

> **iOS・Android での AR 動作確認には HTTPS が必要です。** ngrok 等でトンネルを作ってください。
> ```bash
> ngrok http 8080
> ```

## 技術構成

- **A-Frame 1.5.0** を CDN から読み込み
- **8th Wall Engine Binary** ([`@8thwall/engine-binary`](https://github.com/8thwall/engine)) を jsDelivr CDN から読み込み
  - SLAM（世界トラッキング）バイナリを含む / App Key 不要 / バイナリ専用ライセンス（無償）
  - `xrconfig` + `xrweb` A-Frame コンポーネントでカメラフィードと AR セッションを管理
- `ar-continuous-hit-test` カスタムコンポーネントが `XR8.XrController.hitTest()` を毎フレーム実行してリティクルを更新
- キャラクターは `a-plane` で表示
- 足元にはキャラクターのシルエットを地面に投影した影を表示している

### 対応環境

| 環境 | AR 動作 |
| --- | --- |
| iOS 16+ Safari | ✅ SLAM（8th Wall） |
| Android Chrome | ✅ SLAM（8th Wall） |
| デスクトップブラウザ | 3D ビュー（フォールバック） |

## 画面と操作フロー

### 1. 初期画面

- キャラクター選択用のプリセットボタンを表示
- ローカル画像アップロードに対応
- 選択中の画像をプレビュー表示
- 画面タップから AR セッション開始を試行

### 2. AR 対応時（iOS・Android）

1. 画面をタップして開始
2. XR8 がカメラパーミッションを要求する（初回のみ）
3. ヒットテストで平面を検出する
4. 画面タップごとに現在選択中のキャラクターを追加配置する
5. 同じキャラクターがすでに配置済みなら、新規追加せず既存のキャラクターを選択する
6. 配置済みキャラクターをタップして編集対象を切り替える
7. 拡大・縮小は選択中のキャラクターに適用される
8. `撮影モード` で UI を隠し、画面タップで通常モードへ戻る

### 3. 非対応時のフォールバック

- XR8 が読み込まれていない場合（デスクトップ等）は 3D ビューに切り替える
- `キャラを追加` ボタンでキャラクターを複数配置できる
- ただし同じキャラクターは 1 体までで、追加しようとすると既存の同キャラクターを選択する
- 配置済みキャラクターのタップ選択、拡大・縮小、撮影モード、終了操作は利用できる

## 実装上のポイント

- 画像読込時にアスペクト比を再計算し、新しく配置するキャラクターの横幅へ反映している
- キャラクターは固定の 1 体ではなく、配置時に `a-entity` を動的生成して追加している
- 各キャラクターは本体、単色の影、選択中インジケーターで構成される
- 配置済みキャラクターはキーで一意管理し、同じキャラクターの再配置時は既存要素を選択状態に切り替える
- 選択中キャラクターはタップ位置と各キャラクターの画面投影位置を比較して判定している
- アップロード画像は `URL.createObjectURL()` を使って読み込み、Blob URL はページ終了時に解放している
- プリセット一覧はグリッド表示で、マウスドラッグによるスクロールに対応している
- UI は固定 `div` オーバーレイとして実装し、`touchstart` 時にタップ先が UI 内かチェックしてキャラクター配置を抑制している
- `戻る` 操作は 8th Wall 使用時は UI リセットのみ（カメラセッションは継続）、フォールバックモードでも初期画面へ戻れる

## 制約と注意点

- ビルドツールやモジュール分割は使っておらず、HTML 1 ファイルにロジックが集約されている
- **8th Wall Engine Binary は MIT ライセンスではなくバイナリ専用ライセンスです。** 商用・非商用ともに無償で利用可能ですが、[ライセンス条文](https://github.com/8thwall/engine/blob/main/LICENSE) を確認してください
- AR 動作には HTTPS が必要（カメラパーミッション要件）
- 撮影モードは UI を隠す機能であり、保存自体は端末の通常スクリーンショットや録画機能を使う想定
- プリセット画像や UI 文言は `index.html` に直接書かれている

## ローカルでの扱い

静的ファイルとして配置するだけで動かせる構成です。  
AR 機能の確認には、**HTTPS 環境と iOS 16+ Safari または Android Chrome** が必要です。
