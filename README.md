# AR Character Viewer

AR でキャラクター立ち絵を表示するシンプルな Web アプリです。  
A-Frame と WebXR を使い、ブラウザ上でキャラクター画像を平面として配置できます。

## 概要

- プリセット画像からキャラクターを選択できる
- ローカル画像をアップロードして差し替えできる
- WebXR AR 対応環境では実空間の平面に複数キャラクターを配置できる
- 非対応環境では 3D ビューにフォールバックして表示できる
- 配置後はタップで編集対象を選び、拡大・縮小できる
- UI を完全に隠す撮影モードを利用できる

## 現在の構成

このアプリは主に `index.html` 1 ファイルで構成されています。

| パス | 役割 |
| --- | --- |
| `index.html` | UI、スタイル、A-Frame シーン、AR 制御ロジックをまとめて実装 |
| `circle.png` | 補助用画像アセット |
| `*.png` | キャラクター立ち絵のプリセット画像 |

## 技術構成

- **A-Frame 1.5.0** を CDN から読み込み
- `a-scene` で **WebXR** を有効化
- `webxr` 属性で `hit-test`, `local-floor`, `dom-overlay` を使用
- キャラクターは `a-plane` で表示
- 影も別の `a-plane` として構成し、黒色・半透明で地面に投影風に見せている

## 画面と操作フロー

### 1. 初期画面

- キャラクター選択用のプリセットボタンを表示
- ローカル画像アップロードに対応
- 選択中の画像をプレビュー表示
- `ARを開始` ボタンから AR セッション開始を試行

### 2. AR 対応時

1. `ARを開始` を押す
2. ブラウザが `immersive-ar` をサポートしていれば AR モードへ入る
3. ヒットテストで平面を検出する
4. 画面タップごとに現在選択中のキャラクターを追加配置する
5. 配置済みキャラクターをタップして編集対象を切り替える
6. 拡大・縮小は選択中のキャラクターに適用される
7. `撮影モード` で UI を隠し、画面タップで通常モードへ戻る

### 3. 非対応時のフォールバック

- WebXR または AR セッションが利用できない場合は 3D ビューに切り替える
- `キャラを追加` ボタンでキャラクターを複数配置できる
- 配置済みキャラクターのタップ選択、拡大・縮小、撮影モード、終了操作は利用できる

## 実装上のポイント

- 画像読込時にアスペクト比を再計算し、新しく配置するキャラクターの横幅へ反映している
- キャラクターは固定の 1 体ではなく、配置時に `a-entity` を動的生成して追加している
- 各キャラクターは本体、影、選択中インジケーターで構成される
- 選択中キャラクターはタップ位置と各キャラクターの画面投影位置を比較して判定している
- アップロード画像は `URL.createObjectURL()` を使って読み込み、Blob URL はページ終了時に解放している
- プリセット一覧はグリッド表示で、マウスドラッグによるスクロールに対応している
- UI は `dom-overlay` 上にあり、撮影モード中は全面的に非表示になる

## 制約と注意点

- ビルドツールやモジュール分割は使っておらず、HTML 1 ファイルにロジックが集約されている
- 回転移動や削除などの編集機能は未実装
- WebXR AR の利用可否はブラウザと端末依存
- WebXR AR では、ブラウザから実写背景込みの画像保存や端末スクリーンショット機能の直接起動は前提にしていない
- 撮影モードは UI を隠す機能であり、保存自体は端末の通常スクリーンショットや録画機能を使う想定
- プリセット画像や UI 文言は `index.html` に直接書かれている

## ローカルでの扱い

静的ファイルとして配置するだけで動かせる構成です。  
AR 機能の確認には、WebXR AR をサポートするブラウザと端末が必要です。
